Quiz 10 — Sample Question

✗ ExplanationWhy 'A reinforcement learning model trained on reward signals' is wrong: Reinforcement learning models learn policies through reward optimization in interactive environments. They are not designed to produce semantic text embeddings and are unrelated to the document vectorization task.

Quiz 12 — Sample Question
A team is comparing BERTScore against other text generation metrics. What distinguishes how BERTScore evaluates output quality?
  • ✓ Contextual embeddings to measure semantic similarity
  • N-gram overlap between generated and reference text
  • Sentence-level perplexity from a language model
  • Weighted n-gram precision like BLEU
Show Explanation
✓ Why correctBERTScore leverages pre-trained BERT contextual embeddings to compute token-level similarity between generated and reference texts, capturing semantic meaning beyond surface-level word overlap. It produces precision, recall, and F1 scores based on embedding cosine similarity.
✗ Why B is wrongROUGE measures n-gram overlap between generated and reference text — it counts shared words and phrases but does not understand meaning. A sentence with different words but identical meaning would score poorly under ROUGE, which is exactly what BERTScore is designed to fix.
✗ Why C is wrongSentence perplexity measures how confidently a language model predicts a sequence of tokens; it reflects fluency, not similarity to a reference text. BERTScore requires a reference to compare against, making perplexity a fundamentally different type of metric.
✗ Why D is wrongBLEU computes weighted n-gram precision between a generated translation and one or more reference translations. Like ROUGE, it relies on exact or near-exact word matches and does not capture semantic equivalence the way BERTScore does with embeddings.
Quiz 13 — Sample Question
A company is building a machine translation system and needs to select an automated evaluation metric to benchmark output quality against reference translations at scale. Which metric is most appropriate?
  • ROUGE
  • ✓ BLEU
  • BERTScore
  • Human evaluation only
Show Explanation
✓ Why correctA company evaluating a machine translation system should use BLEU (Bilingual Evaluation Understudy), which measures n-gram overlap between the model's output and reference translations. It is the standard benchmark metric for translation tasks and is widely used across AWS documentation and ML literature.
✗ Why A is wrongROUGE (Recall-Oriented Understudy for Gisting Evaluation) is designed for summarization tasks—it measures recall of overlapping n-grams against reference summaries. Using ROUGE for translation would not accurately reflect translation quality.
✗ Why C is wrongBERTScore uses contextual embeddings to measure semantic similarity between generated and reference text. While useful for semantic evaluation, it is not the primary standard metric for translation benchmarking.
✗ Why D is wrongHuman evaluation (having reviewers rate translations) is valuable but expensive, slow, and not scalable. It is impractical as the primary automated evaluation metric for a production translation system.

Quiz 16 — Sample Question
A team is evaluating tradeoffs in AI system design. Which THREE represent real tradeoffs that practitioners must actively manage?
  • ✓ Interpretability versus performance (accuracy)
  • Fairness and safety always improve together with no conflict
  • Adding more training data always reduces both bias and variance equally
  • Increasing model complexity always improves generalization
  • ✓ Privacy versus model utility
  • ✓ Safety versus transparency in some contexts
Show Explanation
✓ Why correct(A) Interpretability vs. performance is a foundational tradeoff — highly accurate deep learning models (e.g., large neural networks) are often black boxes, while simpler, interpretable models like decision trees sacrifice some accuracy for explainability. This directly affects regulated use cases. (E) Privacy vs. model utility: techniques that protect user privacy — such as differential privacy, data anonymization, or federated learning — reduce the signal available to the model, lowering predictive performance. Teams must balance the obligation to protect personal data against the model's ability to deliver accurate results. (F) Safety vs. transparency: publishing full model details (weights, training data) improves external auditability but can expose security vulnerabilities or be misused to craft adversarial inputs — a real tension documented in responsible AI frameworks.
✗ Why B is wrongFairness and safety can conflict. For example, adding strict safety filters may disproportionately restrict outputs for certain demographic groups, inadvertently reducing fairness. They must be balanced, not assumed to improve together.
✗ Why C is wrongAdding more training data reduces variance (overfitting) but does not equally reduce bias (underfitting). Bias is primarily a function of model architecture and assumptions, not dataset size. This is a common misconception about the bias-variance tradeoff.
✗ Why D is wrongIncreasing complexity generally reduces training error but can increase variance and reduce generalization on unseen data — a core concept of the bias-variance tradeoff. More complexity does not always improve generalization.

Quiz 23 — Sample Question
A company's AI chatbot repeatedly receives identical customer-support questions. They want to reduce both inference latency and model invocation costs without changing their model. What is the primary benefit of adding a caching layer in front of their inference endpoint?
  • ✓ Reduces inference costs and latency for repeated queries
  • Slows down responses by adding a caching layer
  • Provides no measurable benefit for AI workloads
  • Increases costs by requiring additional infrastructure
Show Explanation
✓ Why correctCaching stores the output of expensive inference calls so that identical or near-identical requests are served from memory instead of invoking the model again. This eliminates redundant compute, directly reducing per-request inference costs, and delivers sub-millisecond response times compared to the hundreds of milliseconds a live model call may require.
✗ Why B is wrongElastiCache or MemoryDB serve cached results from DRAM at microsecond to low-millisecond latency—far faster than querying a model or a relational database. Caching always improves response times for repeated queries, never degrades them.
✗ Why C is wrongFor workloads with repeated or predictable queries (e.g., the same user question asked thousands of times), caching delivers measurable cost savings and latency improvements. The benefit scales with cache hit rate.
✗ Why D is wrongCaching reduces inference costs because the model is not invoked for cache-hit requests. Foundation model inference on services like Amazon Bedrock or SageMaker endpoints is billed per token or per invocation—avoiding those calls directly cuts the bill.

  • Rely on VPC security groups alone to protect S3 data without enabling bucket-level controls
Show Explanation
✗ Why options B, C, D are correctS3 Versioning (B) maintains a full history of every object version, enabling recovery from accidental overwrites or deletions by restoring a prior version. Default server-side encryption (C) using SSE-S3 (AES-256 managed by Amazon) or SSE-KMS (customer-managed keys via AWS KMS) ensures data is encrypted before it is written to disk, protecting confidentiality even if storage-layer access controls are bypassed. IAM policies and bucket policies (D) enforce the principle of least privilege, ensuring only authorized SageMaker execution roles, data engineers, and pipelines can read or write training data. Together these three controls address confidentiality (encryption), access integrity (IAM/bucket policies), and recoverability (versioning).
✗ Why A is wrongS3 Transfer Acceleration is a network performance feature that routes uploads through CloudFront edge locations to improve upload speed from geographically distant clients. It has no effect on the security or protection of data already stored in S3. Enabling or disabling it does not change who can access, overwrite, or delete your training data.
✗ Why E is wrongUsing a single shared bucket with no object-level policies violates the principle of least privilege. Different teams and automated pipelines should be restricted to only the specific prefixes or objects they require. A flat-permission bucket means a misconfigured or compromised service can read, overwrite, or delete any training dataset in the bucket.
✗ Why F is wrongVPC security groups control network-level access to EC2 instance network interfaces — they do not protect objects stored in Amazon S3. S3 is accessed via HTTPS API calls that are authorized by IAM and bucket policies, not by security groups. S3 Block Public Access, bucket policies, and IAM policies are the correct controls for S3 data protection.

  • ✓ They represent items as numerical vectors in a shared space where similar items are positioned closer together

Show Explanation
✓ Why correctEmbeddings map data (text, images, products) into dense numerical vectors in a high-dimensional space. In this space, semantically similar items are positioned closer together, which makes cosine similarity or nearest-neighbor searches effective for finding related products. This is the core reason embeddings power recommendation and search systems.
✗ Why A is wrongEmbeddings are dense continuous vectors, not binary category labels. Category labels are a separate concept from the learned semantic representations that embeddings provide—they cannot capture nuanced similarity between items.

Embeddings capture semantic similarity regardless of purchase frequency, meaning they can surface relevant items that are rarely purchased but closely related to the customer's interests.
